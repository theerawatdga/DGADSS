ภาครัฐ

คู่มือการออกแบบและพัฒนาบริการดิจิทัลที่สอดคล้องตามประกาศ DGA

📱 ประเภทแอปพลิเคชัน (Application Type)

1. Website / Web Application

เน้นการให้ข้อมูล ประชาสัมพันธ์ รองรับการค้นหา (SEO) เข้าถึงผ่านเบราว์เซอร์เป็นหลัก การทำงานและทำธุรกรรม มีระบบล็อกอิน มีความซับซ้อนของ Logic หลังบ้าน และการจัดการฐานข้อมูล

2. Mobile Application

มีการใช้งานบนสมาร์ตโฟน ต้องการใช้ฟีเจอร์ของเครื่อง (เช่น กล้อง, GPS, Push Notification) ติดตั้งผ่าน Store

🔄 ประเภทบริการ (Service Type)

1
บริการข้อมูลข่าวสาร (Informational Service)

การสื่อสารทางเดียว ไม่มีการทำธุรกรรม

2
บริการธุรกรรมกึ่งดิจิทัล (Partial Digital Transactional Service)

ยังต้องทำกิจกรรมทางกายภาพบางส่วน

3
บริการธุรกรรมดิจิทัลเบ็ดเสร็จ (Fully Digital Transactional Service)

ครบทุกขั้นตอน End-to-End ผ่านออนไลน์

4
พอร์ทัลกลาง (Portal)

One-stop Service ศูนย์กลางเชื่อมโยงหลายหน่วยงาน

พัฒนาโดยสำนักงานพัฒนารัฐบาลดิจิทัล (องค์การมหาชน)

www.dga.or.th อ่านประกาศฉบับเต็ม (PDF)

การจำแนกประเภทบริการ

เลือกประเภทแอปพลิเคชันและประเภทบริการของท่าน เพื่อให้ระบบกรองมาตรการควบคุมที่เกี่ยวข้องโดยอัตโนมัติ

การประเมินระดับผลกระทบ (Impact Level)

อ้างอิงตามประกาศคณะกรรมการการรักษาความมั่นคงปลอดภัยไซเบอร์แห่งชาติ เรื่อง มาตรฐานการกำหนดคุณลักษณะความมั่นคงปลอดภัยไซเบอร์ พ.ศ. 2566

1. มูลค่าความเสียหายทางการเงินหรือทรัพย์สินหรือต่อชื่อเสียงของหน่วยงาน
2. จำนวนของผู้ใช้บริการของหน่วยงาน บุคลากรของหน่วยงาน หรือประชาชนที่อาจได้รับอันตรายต่อชีวิต ร่างกาย อนามัย ทรัพย์สิน หรือความเสียหายอื่นใด
3. ความสามารถในการดำเนินการตามหน้าที่ของหน่วยงาน
4. ความมั่นคงของรัฐและความสงบเรียบร้อยภายในประเทศ
